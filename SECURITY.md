# Security Policy

## Supported Versions

Security updates are provided for the latest published version on pub.dev.

| Version | Supported |
| ------- | --------- |
| Latest  | Yes       |
| Older   | No        |

## Reporting a Vulnerability

If you find a security issue, please report it privately and avoid posting
public exploit details.

Preferred reporting path:

- Open a private security advisory draft:
  https://github.com/dedinopriadi/bee_dynamic_launcher/security/advisories/new

If private advisories are unavailable in your repository settings:

- Open an issue marked with `security` and share only high-level impact details.
- A maintainer will follow up for private reproduction and patch validation.

## Response Process

The maintainers will:

1. Acknowledge receipt of the report.
2. Validate impact and affected versions.
3. Prepare and test a fix.
4. Publish the fix and coordinate disclosure notes.

Response time depends on report complexity and maintainer availability.
